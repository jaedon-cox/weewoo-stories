# AutoStory — Autonomous Reddit Story TikTok Pipeline

## Project Overview

AutoStory is a fully automated content pipeline that generates Reddit-style stories using AI, converts them to voiceover audio, composites them into short-form videos with background footage and captions, splits them into platform-optimized parts, and posts them on a scheduled cadence to TikTok and YouTube Shorts — with zero manual involvement after initial setup.

---

## Goals

- Generate engaging Reddit-style story content autonomously (no human-written prompts)
- Produce polished short-form videos with voiceover, captions, and background footage
- Split long videos into parts of at least 60 seconds each
- Post to TikTok and YouTube Shorts on a scheduled cadence
- Run entirely on a cloud VPS with no dependency on a local machine
- Keep monthly operating costs under $30

---

## Pipeline Stages

### 1. Prompt Generation
- Claude selects its own topic, subreddit style, drama level, and character names
- Rotates through a weighted pool of categories: AITA, TIFU, r/relationship_advice, r/prorevenge, etc.
- Outputs a structured prompt object: `{ category, title, tone, word_count_target }`

### 2. Story Writing
- Claude API generates a full Reddit-style story from the prompt object
- Target length: 600–1200 words (approximately 3–6 minutes of audio)
- Story is saved to the database with a unique ID and status `pending_tts`

### 3. Text-to-Speech
- ElevenLabs API converts the story text to a natural-sounding voiceover MP3
- Configurable voice ID per category or account
- Fallback: Google Cloud TTS if ElevenLabs quota is exceeded
- Output saved to `/media/audio/{story_id}.mp3`

### 4. Video Composition
- FFmpeg overlays the voiceover audio onto a randomly selected background video clip
- Background clips loop if the audio is longer than the clip
- Burned-in subtitles generated from the TTS transcript using `ffmpeg subtitles` filter
- Output: a single composite video file at 1080x1920 (TikTok/Shorts portrait format)
- Output saved to `/media/composed/{story_id}.mp4`

### 5. Video Splitting
- FFmpeg `-segment_time` splits the composed video into parts of 60–90 seconds each
- Each part named `{story_id}_part{n}.mp4`
- Parts registered in the database with status `queued` and a `scheduled_at` timestamp

### 6. Upload Queue & Scheduler
- SQLite database tracks all parts with the following states: `queued → uploading → posted → failed`
- APScheduler runs on a configurable cadence (default: 3 posts/day)
- Generator job runs on a configurable cadence (default: 5 new stories/week)
- Failed uploads are retried up to 3 times with exponential backoff

### 7. TikTok Posting
- TikTok Content Posting API v2 (official API, not browser automation)
- OAuth2 authentication with stored refresh token
- AI-generated content label applied to every post (TikTok compliance)
- Caption and hashtags auto-generated by Claude based on story title and category
- Token expiry monitor emails the operator a re-auth link 7 days before expiration

### 8. YouTube Shorts Posting
- YouTube Data API v3 uploads the same video parts to a connected YouTube channel
- Description and tags auto-generated alongside TikTok caption
- Scheduled as unlisted → public on the same cadence as TikTok

## Tech Stack

| Component | Tool |
|---|---|
| Story generation | Claude API (claude-sonnet-4-20250514) |
| Text-to-speech | ElevenLabs API (fallback: Google Cloud TTS) |
| Video composition | FFmpeg |
| Video processing helper | moviepy (optional) |
| Database / queue | SQLite |
| Scheduler | APScheduler |
| TikTok posting | TikTok Content Posting API v2 |
| YouTube posting | YouTube Data API v3 |
| Process management | PM2 or systemd |
| Hosting | DigitalOcean Droplet ($6/mo) or equivalent VPS |
| Storage | Local VPS storage (background clips + generated media) |

## Configuration (`config.yaml`)

```yaml
posting:
  tiktok_enabled: true
  youtube_enabled: true
  posts_per_day: 3
  post_times: ["12:00", "18:00", "22:00"]

generation:
  stories_per_week: 5
  min_word_count: 600
  max_word_count: 1200
  categories:
    - name: AITA
      weight: 0.35
    - name: TIFU
      weight: 0.25
    - name: relationship_advice
      weight: 0.20
    - name: prorevenge
      weight: 0.20

video:
  resolution: "1080x1920"
  min_part_duration_seconds: 60
  max_part_duration_seconds: 90
  subtitle_font: "Arial"
  subtitle_size: 18

tts:
  provider: "elevenlabs"       # or "google"
  elevenlabs_voice_id: "YOUR_VOICE_ID"
  fallback_provider: "google"

alerts:
  token_expiry_warning_days: 7
  alert_email: "you@example.com"
```

## Database Schema

```sql
CREATE TABLE stories (
  id TEXT PRIMARY KEY,
  category TEXT,
  title TEXT,
  body TEXT,
  status TEXT,  -- pending_tts | pending_video | pending_split | done | failed
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE parts (
  id TEXT PRIMARY KEY,
  story_id TEXT REFERENCES stories(id),
  part_number INTEGER,
  file_path TEXT,
  status TEXT,  -- queued | uploading | posted | failed
  scheduled_at DATETIME,
  posted_at DATETIME,
  tiktok_video_id TEXT,
  youtube_video_id TEXT,
  retry_count INTEGER DEFAULT 0
);
```

## Monetization Path

- **TikTok Creator Reward Program** — requires 10k followers + 100k views/30 days to qualify. Pays ~$0.40–$0.80 per 1,000 views.
- **YouTube Shorts Fund / AdSense** — typically 3–5x better RPM than TikTok for this content type.
- **Break-even point** — approximately 50,000–75,000 combined views/month.
- **Brand deals** — realistic at 50k–100k followers, $200–$1,000+ per sponsored post.

---

## Compliance Notes

- All posts must use TikTok's **AI-generated content label**
- Automated *posting* via the official API is permitted; automated liking/following/commenting is not
- Occasional manual account engagement is recommended to maintain account health
- YouTube requires disclosure of AI-generated content in descriptions
- TikTok OAuth2 tokens expire every 30–90 days and require a manual re-auth (token expiry alert will email you in advance)

## Setup Steps (one-time)

1. Provision a VPS (Ubuntu 22.04 recommended)
2. Install Python 3.11+, FFmpeg, PM2
3. Clone repo and run `pip install -r requirements.txt`
4. Add API keys to `.env`
5. Upload background video clips to `media/backgrounds/`
6. Run `python db/setup.py` to initialize the database
7. Authenticate TikTok and YouTube OAuth2 (`python auth/tiktok_auth.py`, `python auth/youtube_auth.py`)
8. Start the orchestrator: `pm2 start main.py --name autostory`
